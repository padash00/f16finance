# Orda Point Operator — журнал изменений

## v2.7.1 — Запуск офлайн (без интернета / при проблемах с сервером)

### Исправлено
- **Главное: программа больше не «зависает» без интернета.** Раньше при
  запуске она обязательно звонила на сервер (`bootstrap`), и если интернета
  не было ИЛИ сервер/Vercel был недоступен — выкидывала на экран входа, где
  офлайн войти нельзя. Теперь: если есть сохранённая сессия и кэш —
  программа **сразу открывается в рабочем режиме офлайн** с кэшированными
  товарами. Продажи копятся в локальную очередь и уходят на сервер, когда
  связь возвращается.
- Оба сценария (нет интернета / лежит сервер) теперь обрабатываются
  одинаково — переход в офлайн-режим, а не блокировка.

## v2.7.0 — Реклама на экране клиента

### Главное
На втором мониторе (экран клиента) во время простоя кассы теперь крутится
рекламный плейлист — видео и картинки. Контент настраивается из веб-админки
(раздел «Магазин» → «Реклама на экране клиента»).

### Что нового
- **Плеер рекламы** в режиме простоя: картинки сменяются по заданной
  длительности, видео проигрывается без звука до конца, затем по кругу.
- Когда оператор начинает продажу — экран клиента переключается на корзину,
  после оплаты показывает «Спасибо», затем снова возвращается к рекламе.
- Часы остаются в углу экрана.
- Плейлист подтягивается при старте и обновляется раз в 10 минут.
- При открытии окна клиента состояние пересылается сразу — реклама
  появляется без задержки.

## v2.4.0 — Новый минималистичный интерфейс продаж

### Главное
Полностью переработана страница продаж под минималистичный стиль (как у Wipon Pro):
большой поиск сверху, таблица позиций по центру, оплата справа.
Адаптивно от 10" планшета до 34" монитора.

### Что нового
- **Поиск товара** во всю ширину сверху с автоматическим добавлением:
  - сканирование штрихкода — точное совпадение добавляется сразу
  - поиск по названию — Enter добавляет первый результат
- **Таблица позиций** в центре экрана: # / Наименование / Цена / Кол-во / Сумма
- **Оплата справа** на больших экранах (или внизу на маленьких):
  - Большое «К оплате» крупным шрифтом
  - Способы оплаты: Наличные / Kaspi / Смешанная
  - **В смешанной обе суммы редактируются и автозаполняются**: ввёл Kaspi 600 → наличные 600 появятся сами
- **Универсальный товар** — кнопка `+ Товар` для разовой продажи того, чего нет в каталоге.
  Указываешь название и цену — попадает в чек и финансовый отчёт.
- **Скидки**: ручной % или промокод
- **Клиенты и баллы лояльности**: поиск, использование баллов
- **Комментарий к продаже**
- **Печать чека** автоматически после оплаты — крупнее и информативнее (80mm термо)
- **Звуковые сигналы** при добавлении (бип) и ошибке (низкий)
- **История продаж** — вкладка с последними чеками за смену
- **Корректировка оплаты** на проведённых чеках (с указанием причины)

### Улучшения скорости
- Кэширование витрины: страница открывается мгновенно при медленном интернете,
  обновление в фоне
- Приложение остаётся работоспособным при обрывах связи

### Исправления (под капотом)
- Касса бьёт строго по витрине (point_display) — без скрытых автотрансферов
- Если на витрине нет товара, продажа отклоняется с понятным сообщением
  «На витрине недостаточно «Кола»: есть 0, нужно 1. Сделайте заявку склад → витрина.»
- Все ошибки переведены на русский с подсказками к действию

### Возврат на старый UI
Если что-то не работает — можно вернуть классический UI:
- На стороне сборки: `VITE_USE_LEGACY_SALES=1 pnpm run release`

### Системные изменения
- Серверная поддержка универсальных товаров (миграция 20260506_point_sale_universal_items.sql)
- Полный рефактор учёта склад/витрина (см. репозиторий, шаги inventory v2)

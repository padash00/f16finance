## 🔴 КРИТИЧНО

### 1. Email суперадмина захардкожен в базе данных
**Файл:** `supabase/migrations/20260313_shifts_rls.sql`
```sql
coalesce(lower(auth.jwt() ->> 'email'), '') in (
  'padash00@gmail.com'   ← твой email прямо в SQL!
)
```
**Проблема:** Email захардкожен в RLS-функции. Если изменишь email — нужна новая миграция. Email виден всем кто смотрит миграции.
**Решение:** Создать таблицу `super_admins` или хранить признак в `auth.users.app_metadata`.

---

### 2. Нет ограничения запросов (Rate Limiting)
**Файлы:** Все `/app/api/admin/*/route.ts`
**Проблема:** Любой авторизованный пользователь может слать тысячи запросов в секунду. Нет защиты на POST-эндпоинты.
**Решение:** Подключить Upstash или простой in-memory rate limiter в middleware.

## 🟠 ВЫСОКИЙ

### 3. N+1 запросы и безлимитные выборки

**`lib/server/repositories/inventory.ts`** — `fetchInventoryOverview` загружает до **300 движений** с вложенными join-ами (location + company). Это 900+ SQL lookup-ов за раз.
```typescript
.limit(300),  // ← слишком много
```

**`app/api/admin/customers/route.ts`** — загружает **всех клиентов**, потом фильтрует в памяти:
```typescript
customers = customers.filter(c => ...)  // ← фильтр должен быть в SQL
```

**`app/api/admin/operators/route.ts`** — Promise.all из 4 запросов за 30 дней без лимита строк. С 10k+ записями — тормоза.

**Решение:** Добавить пагинацию + фильтрацию на уровне SQL во всех этих местах.

---

### 4. Нет пагинации на больших таблицах
- Клиенты — нет лимита, нет страниц
- Движения инвентаря — лимит 300, но нет кнопки "загрузить ещё"
- Расходы/доходы — возвращают всё за период без paging

---

### 5. `alert()` вместо нормального UI
**Файлы:** `app/(main)/access/page.tsx`, `app/(main)/customers/page.tsx`, `app/(main)/store/warehouse/page.tsx`
```typescript
alert(err?.message || 'Ошибка удаления')  // ← блокирует весь UI
alert('Ошибка сети')
```
**Решение:** Заменить на `toast.error()` — Sonner уже подключен в проекте.
> **Быстрая правка: 30 минут, огромный эффект на UX**

---

### 6. Нет проверки входных данных (Input Validation)
**Файл:** `app/api/admin/expenses/route.ts`
```typescript
const sort = url.searchParams.get('sort') as 'date_desc' | 'date_asc'  // ← cast без проверки
```
**Решение:** Добавить Zod-валидацию на все query-параметры и тела запросов.

---

### 7. Sandbox отключен в Electron
**Файл:** `desktop/operator/electron.cjs`
```typescript
sandbox: false,  // ← небезопасно
```
**Проблема:** Если renderer скомпрометирован — атакующий получает доступ к Node.js API.
**Решение:** `sandbox: true` + `contextIsolation: true` + `nodeIntegration: false`

---

### 8. Нет автовосстановления при белом экране в киоске
**Файл:** `desktop/kiosk/main/main.js`
**Проблема:** Если renderer зависает или падает — белый экран навсегда до перезапуска.
**Решение:**
```javascript
mainWindow.webContents.on('crashed', () => {
  mainWindow.webContents.reload()
})
mainWindow.webContents.on('unresponsive', () => {
  mainWindow.loadURL(serverBaseUrl)
})
```
> **Это скорее всего и есть причина текущего белого экрана!**

---

### 9. Нет оффлайн-режима в operator desktop
**Проблема:** Очередь (`queue.json`) уже начата, но автоматической синхронизации при восстановлении сети нет. Оператор не может работать при падении интернета.

---

### 10. CORS-заголовки в Electron — слишком открытые
**Файл:** `desktop/operator/electron.cjs`
```javascript
'Access-Control-Allow-Origin': ['*'],  // ← открыто для всех!
```
**Решение:** Заменить `*` на конкретные разрешённые домены.

## 🟡 СРЕДНИЙ

### 11. Дублирование кода — canManage / canView

Одна и та же функция скопирована в 8+ файлах:
```typescript
// Повторяется в каждом route.ts:
function canManage(access) {
  return access.isSuperAdmin || access.staffRole === 'owner' || access.staffRole === 'manager'
}
```
**Решение:** Вынести в `lib/server/access-control.ts` один раз.

---

### 12. Дублирование — normalizeQty / normalizeMoney

Функция нормализации числа скопирована в 5+ файлах:
```typescript
function normalizeQty(v: unknown) {
  const n = Number(v || 0)
  return Number.isFinite(n) ? Math.round((n + Number.EPSILON) * 1000) / 1000 : 0
}
```
**Решение:** Вынести в `lib/server/money.ts`.

---

### 13. Нет индексов на часто используемых колонках

Нужно добавить миграцию с индексами:
```sql
-- Для фильтрации остатков склада
create index idx_inventory_balances_item_location
  on inventory_balances(item_id, location_id) include (quantity);

-- Для выборки доходов по оператору и дате
create index idx_incomes_operator_date
  on incomes(operator_id, date desc);

-- Для движений по складу
create index idx_inventory_movements_from_to_date
  on inventory_movements(from_location_id, to_location_id, created_at);
```
> **Даст ускорение запросов в 5-10 раз**

---

### 14. Нет RLS на некоторых таблицах инвентаря

Таблицы `inventory_balances`, `inventory_movements` могут не иметь Row Level Security. Это значит авторизованный пользователь может читать данные любой компании.

---

### 15. ~1000+ вхождений `any` в TypeScript
```typescript
const { data, error } = await (supabase as any).from(...)
```
Потеря типобезопасности. Ошибки в именах колонок не поймаешь на этапе компиляции.

---

### 16. Нет debounce на поиске
**Файл:** `app/(main)/inventory/page.tsx`
```typescript
<CommandInput onValueChange={setSearch} />  // ← каждый символ = перерисовка
```
С 5000+ товаров — тормозит. Добавить `useDebouncedCallback(fn, 300)`.

---

### 17. Нет виртуального скролла на больших списках

`@tanstack/react-virtual` **уже есть в зависимостях**, но не используется. При 1000+ строках без него — тормоза и зависания.

---

### 18. Дублирование блока catch во всех API routes
```typescript
// Повторяется в каждом route.ts:
} catch (error: any) {
  await writeSystemErrorLogSafe({ ... })
  return json({ error: error?.message }, 500)
}
```
**Решение:** Обертка `withApiHandler(fn)` — одна строка вместо 4.

## 🟢 НИЗКИЙ

### 19. Хардкоженный путь к игре
**Файл:** `desktop/kiosk/main/main.js`
```typescript
'D:\\Games\\CS2\\cs2.exe'  // ← диск D может не существовать
```

### 20. Медленное определение оффлайна
5 consecutive failures до детекции (25+ секунд). Снизить до 2-3 или использовать `navigator.onLine`.

### 21. Нет оптимистичных обновлений
Юзер жмёт "Сохранить" → ждёт 1-2 сек → видит результат. Можно обновлять UI мгновенно, откатывать при ошибке.

### 22. Нет горячих клавиш
Ctrl+S для сохранения, Ctrl+K для поиска. Удобно для менеджеров.

### 23. Нет печати чеков/накладных
Приходы, продажи, накладные нельзя распечатать. Нужен шаблон + `window.print()`.

### 24. Нет экспорта в Excel с фильтрами
Экспорт есть, но без учёта текущих фильтров. Выгружает всё, а не то что видит пользователь.

### 25. `legacy-redirect.tsx` — мертвый код
Судя по названию, можно удалить.

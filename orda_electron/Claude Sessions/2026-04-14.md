## Структура проекта Orda

### Три компонента системы

| Компонент | Папка | Технология |
|-----------|-------|-----------|
| Сайт (веб-платформа) | `/` (корень) | Next.js |
| Операторская программа | `desktop/operator/` | Electron + Vite/React |
| Клиентская программа (kiosk) | `desktop/kiosk/` | Electron (kiosk-режим) |

### Что удалили
- `desktop/orda_point_client/` — старая Python/tkinter операторская программа
- `desktop/client/` — старая версия киоска (конфиг жёстко зашит, нет setup)
- `desktop/orda-kiosk/` → переименована в `desktop/kiosk/`
- `desktop/electron_point/` → переименована в `desktop/operator/`

### Как работает kiosk
1. Первый запуск: `electron . --setup` → окно настройки (stationCode, serverBaseUrl, provisioningKey)
2. Регистрация через `POST /api/kiosk/register` → получает clientSecret, deviceToken
3. Конфиг сохраняется локально (config-store.js)
4. Обычный запуск: kiosk-режим, fullscreen, блокировка Alt+F4/Alt+Tab
5. Связь с сервером: WebSocket + HTTP heartbeat каждые 10 сек
6. IP и MAC адрес физически привязываются к станции на сайте

## План доработки клиентской программы (Kiosk Shell)

### Концепция
Полноценный игровой shell типа Senet/SmartShell/Gizmo.
Клиент видит красивый интерфейс, управляет сессией сам (если есть аккаунт)
или через оператора (гостевой режим).

---

### Два сценария входа

#### Сценарий A — с аккаунтом
1. Клиент приходит → оператор пополняет баланс на сайте
2. Клиент садится → на экране приветствия вводит логин/пароль или сканирует QR
3. Авторизуется → видит свой баланс
4. Покупает тариф сам → сессия запускается
5. Если не может → оператор покупает тариф за него из своей панели

#### Сценарий B — гость (без аккаунта)
1. Клиент приходит → оператор выбирает тариф и активирует станцию
2. Сессия запускается без входа в аккаунт
3. Клиент видит время и каталог игр

---

### Экраны киоска

#### 1. Экран приветствия (станция свободна)
- Красивый fullscreen фон (настраивается с сайта: изображение/видео/цвет/градиент)
- Логотип клуба (настраивается с сайта)
- Два варианта входа по центру:
  - `Войти по логину/паролю` (форма)
  - `Войти по QR-коду` (сканирование телефоном)
- Снизу: бегущая строка / анонсы / часы
- Всё оформление редактируется в разделе "Станции" на сайте

#### 2. Экран выбора тарифа (после входа с аккаунтом)
- Имя игрока, аватар, баланс
- Список доступных тарифов (цена, длительность)
- Кнопка "Купить и начать"
- Если баланс недостаточен → подсказка обратиться к оператору

#### 3. Главный экран — Shell (активная сессия)
- Верхняя панель: логотип, имя игрока / "Гость", тариф, таймер обратного отсчёта
- Каталог контента в виде красивой сетки с обложками:
  - Вкладки: **Игры** | **Браузер** | **Программы**
  - Каждый элемент: большая обложка + название + кнопка "Запустить" снизу
  - Контент редактируется с сайта (добавить/убрать игру, загрузить обложку, путь к .exe)
- Кнопки в углу: Продлить / Вызвать оператора
- Предупреждения: за 10 мин, 5 мин, 1 мин до конца сессии

#### 4. Экран профиля (только для авторизованных)
- Фото аккаунта (можно сменить)
- Смена пароля
- История сессий
- Текущий баланс
- Статистика (часы, любимые игры)

#### 5. Экран завершения сессии
- "Время истекло"
- Кнопки: Продлить / Выйти из аккаунта

#### 6. Экран блокировки
- Если IP/MAC не совпадает с привязкой
- Вызов оператора

│  users, sessions, tariffs, stations,    │

- Никакой отдельной БД, всё через единый API сайта

### Безопасность kiosk-приложения

**Что заблокировано на уровне Electron:**
- Alt+F4, Ctrl+W, Alt+Tab, Ctrl+Shift+Esc, Ctrl+Shift+Del, Win-key (Meta)
- `mainWindow.on('close', e => e.preventDefault())` — нельзя закрыть
- `before-input-event` в webContents блокирует горячие клавиши
- `setKiosk(true)` + `setAlwaysOnTop(true, 'screen-saver')`

**Восстановление сессии после краша/перезагрузки:**
- Heartbeat возвращает `activeSession: { endsAt, tariffName }` если сессия активна
- При старте кiоска: если heartbeat видит активную сессию → сразу восстанавливает таймер

**Watchdog (Windows):**
- Scheduled Task `OrdaKioskWatchdog` — PowerShell-цикл проверяет каждые 10с, жив ли процесс
- Если упал — перезапускает автоматически

**Примечание:** Alt+Tab на уровне приложения заблокировать нельзя (Windows kernel). Нужен GPO или AutoHotKey системно.

### Исправление: kiosk не реагировал на команды оператора

**Корень проблемы:** `stationId` не сохранялся в `config.json` после регистрации.

Каждый heartbeat искал станцию через `.or('station_code.eq.X,name.eq.X')` — мог найти не ту (если в БД похожие коды/имена). Broadcast от оператора шёл на `kiosk:{правильный-uuid}`, кiosk подписывался на `kiosk:{другой-uuid}` → команды не доходили.

**Три исправления в** `desktop/kiosk/main/main.js`:

1. **Сохраняем `stationId`** из ответа `/api/kiosk/register` в config.json
2. **Heartbeat отправляет `stationId` напрямую** (вместо stationCode) → точное совпадение
3. **Realtime инициализируется сразу при старте** (если stationId уже есть в конфиге), не ждёт 10 сек первого heartbeat
4. **Retry realtime**: если `initRealtime` упал (rtconfig недоступен) → `realtimeReady = false` → следующий heartbeat повторит попытку

> **Важно:** существующие зарегистрированные кiоски нужно **перерегистрировать** (кнопка «Сбросить конфиг» в setup screen), чтобы stationId записался в config.json.
